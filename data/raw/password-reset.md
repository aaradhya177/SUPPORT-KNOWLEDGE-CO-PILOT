# Password Reset Guide

## When customers should reset a password

Customers should reset their password when they cannot sign in, suspect that their credentials were shared, or receive a security notice from Northstar Desk. A password reset is also recommended after a team administrator removes a user from a shared device or after a user changes roles and needs to refresh access policies. The reset process does not delete saved tickets, billing records, notification preferences, or team membership. It only replaces the credential used for account sign-in.

## Self-service reset steps

From the sign-in page, select Forgot password and enter the email address associated with the account. Northstar Desk sends a reset link when the address matches an active user. The link expires after thirty minutes and can be used only once. Customers should open the most recent email because requesting another reset invalidates older links. After choosing a new password, the customer is signed out of other browser sessions within a few minutes.

## Troubleshooting missing reset emails

If the customer does not receive the message, ask them to check spam, junk, quarantine, and any company mail filtering tools. They should also confirm that their team administrator has not disabled the account. Support agents can verify whether a reset email was generated, but agents must never ask for the customer's password or create a password on the customer's behalf. If the mail system blocked delivery, the administrator may need to allow messages from notifications@northstardesk.example before the customer requests a new link.

## Security notes for support agents

When handling password issues, verify the requester using the account email and a recent ticket or invoice reference. Do not disclose whether an email belongs to a specific company unless the requester has already authenticated. If there are signs of account takeover, escalate to the security queue and recommend that the administrator reviews audit logs, active sessions, and recently created API keys.
